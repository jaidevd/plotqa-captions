# PlotCaptions

A dataset of over 18 million natural-language captions for scientific plots,
derived from the [PlotQA](https://github.com/NiteshMethani/PlotQA/blob/master/PlotQA_Dataset.md)
dataset. Each caption is a sentence generated by converting a question-answer
pair about a plot into a declarative statement.

For example, given the QA pair:

> **Q:** What is the percentage of female population who survived till age of 65 in 1993?
> **A:** 87.4244

The corresponding caption is:

> The percentage of female population who survived till age of 65 in 1993 is 87.4244.

## Download

The dataset can be **[downloaded here](https://drive.google.com/drive/folders/1DaJhcUXZ5ezQAU_oUrz_uuzLsnu0p6Ob?usp=sharing)**.

The folder contains three files:

| File | Contents |
|---|---|
| `png.tar.gz` | ~150,000 chart images (PNG) |
| `annotations.tar.bz2` | JSON specs of the charts and annotated bounding boxes for QA pairs |
| `captions.tar.bz2` | ~18M captions in JSONL format |

## Dataset format

Each line in the captions JSONL file is a JSON object with the following fields:

```json
{"_id": 1234, "caption": "The percentage of female population who survived till age of 65 in 1993 is 87.4244."}
```

The `_id` field is the image index, corresponding to the PNG filenames and the
annotations file. A single image can have up to a few hundred captions
associated with it.

### Loading the data

```python
import pandas as pd

captions = pd.read_json("captions.jsonl", lines=True)
print(captions.head())
#       _id                                            caption
# 0    1234  The percentage of female population who surviv...
# 1    1234  Across all years, the maximum percentage of ma...
# ...
```

### Matching captions to images

```python
import os
from PIL import Image

image_dir = "png/"
image_id = 1234

# All captions for this image
subset = captions[captions["_id"] == image_id]

# The corresponding chart
img = Image.open(os.path.join(image_dir, f"{image_id}.png"))
```

## How the dataset was created

PlotCaptions is built from the [PlotQA](https://arxiv.org/pdf/1909.00997.pdf)
dataset by Methani et al. The original dataset contains millions of QA pairs
about scientific plots, categorized into three types:

1. **Structural understanding** — questions about the appearance of the plot
   (legend labels, axis titles, etc.)
2. **Data retrieval** — questions about a single data point in the plot.
3. **Reasoning** — questions requiring numeric or visual reasoning.

Since structural questions do not describe the data in a plot, they are filtered
out. The remaining QA pairs are converted to captions through a four-step
pipeline:

### Step 1: Template matching

Each question is matched against one of 49 regex templates defined in
`qa_templates.yaml`. Named capture groups extract the semantic components
(e.g. `yvalue`, `xvalue`, `preposition`) from the question string:

```python
import re

question = "What is the percentage of female population who survived till age of 65 in 1993?"
pattern = r"What is the (?P<yvalue>.*?) (?P<preposition>of|in) (?P<xvalue>.*?)\s?\?$"
matches = re.search(pattern, question).groupdict()
# {'yvalue': 'percentage', 'preposition': 'of',
#  'xvalue': 'female population who survived till age of 65 in 1993'}
```

### Step 2: Caption generation

The extracted groups and the answer are combined using randomized sentence
templates (defined in `captions.py`) to produce varied phrasings:

```python
answer_template = "The {yvalue} {preposition} {xvalue} is {answer}."
caption = answer_template.format(answer=87.4244, **matches)
# "The percentage of female population who survived till age of 65 in 1993 is 87.4244."
```

### Step 3: Text fixes

Automated corrections are applied to all captions (`fixes.py`):

- **Spelling**: `resorces` -> `resources`, `emissisons` -> `emissions`
- **Abbreviations**: `gdp` -> `GDP`, `ppp` -> `PPP`, `imf` -> `IMF`
- **Diacritics**: `Sao Tome` -> `São Tomé`, `Curacao` -> `Curaçao`
- **Determiners**: `Bahamas` -> `the Bahamas`, `United States` -> `the United States`
- **Compounds**: `self employed` -> `self-employed`, `tax payers` -> `taxpayers`
- **Repeated words**, HTML entities, whitespace normalization, and more

### Step 4: Grammar checking

Captions are checked against [LanguageTool](https://languagetool.org/) for
grammatical errors. Subject-verb agreement is corrected using spaCy's
dependency parser (`grammar.py`). Captions with unfixable errors (broken
brackets, NaN values, etc.) are removed.

This process reduced grammar errors from ~15.9 million to near zero.

## Code overview

To reproduce or modify the dataset, see the source modules:

| Module | Purpose |
|---|---|
| `plotqa.py` | Template matching (`match_and_store`) and caption generation (`generate_and_store`) |
| `captions.py` | Randomized caption templates for each of the 49 question types |
| `fixes.py` | Structural question filtering and text corrections (`fix_captions_in_mongo`) |
| `grammar.py` | LanguageTool grammar checking (`check_and_store`) and spaCy-based fixes |
| `qa_templates.yaml` | The 49 regex templates used to match questions |

The pipeline uses MongoDB as intermediate storage. All steps are idempotent.
See the source files for details on prerequisites (MongoDB, LanguageTool,
Python dependencies) and usage.
